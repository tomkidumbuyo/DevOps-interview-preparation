# Audit of `hammadhaqqani/devops-interview-handbook`

Reviewed repository: <https://github.com/hammadhaqqani/devops-interview-handbook>

The public handbook was inspected file-by-file. It contributes a useful conventional DevOps baseline:

- 134 core questions: AWS (20), CI/CD (20), Kubernetes (20), Linux (16), networking (20), security (18), Terraform (20);
- five procedural exercises: deployment failure, Terraform drift, Kubernetes troubleshooting, CI optimization, and incident response;
- three architecture exercises: highly available web app, CI/CD, and multi-region DR;
- junior, mid-level, and senior checklists plus 30/60/90-day roadmaps.

## Themes incorporated

- Definitions and practical comparisons for core AWS, Kubernetes, Terraform, Linux, networking, security and CI/CD concepts.
- Scenario structure: problem/environment/symptoms, ordered diagnosis, root cause, prevention and follow-ups.
- Architecture structure: requirements, assumptions, design, component trade-offs, security, cost, DR and implementation phases.
- Level differentiation and active practice rather than answer memorization.

## Material deliberately expanded

The source handbook is conventional DevOps material. This repository adds the depth needed by the role:

- GPU hardware/software, Kubernetes accelerator operations, scheduling, telemetry and capacity;
- Pulumi, GCP/Vertex AI, AWS Bedrock/SageMaker/EKS AI, and cross-cloud portability;
- model serving, vLLM/KServe/Triton, LLM gateways, RAG and evaluation infrastructure;
- MLOps/LLMOps, agents/tools, AI security, governance, EU regulation and privacy operations;
- SaaS/private-cloud/on-prem/hybrid/air-gapped deployment and customer support;
- AI FinOps, token/GPU unit economics, chargeback, budgets and quality-aware optimization;
- senior platform ownership, product thinking, 30/60/90-day takeover, and behavioural preparation.
