## Junior — normal conceptual and code questions

### VPC-ENDPOINTS-AND-AWS-PRIVATELINK-JN-01

- [ ] **Question:** What is Gateway endpoint, and why does it matter in VPC endpoints and AWS PrivateLink?
> **Covered in:** [VPC endpoints and AWS PrivateLink — Gateway endpoint](README.md#gateway-endpoint)

**Answer:** **Definition.** Route-table target for supported services such as S3/DynamoDB without endpoint ENIs. **Mechanism.** A real request crosses several independent states: name resolution returns an address, the source selects a route and source address, link or overlay forwarding reaches the next hop, stateful or stateless policy evaluates the flow, transport establishes communication, and TLS/application protocols negotiate their own contract. The return path must also work. Its guarantee applies only at its own boundary, so upstream identity, downstream dependencies and asynchronous convergence still matter. **Evidence and failure behavior.** Healthy evidence progresses layer by layer: correct name/address, expected route, permitted flow, listening endpoint, successful handshake and valid application response. Timeouts, refusals, resets and protocol errors mean different layers; packet loss, MTU, asymmetric paths, connection-state exhaustion and proxy timeout mismatch are common production failures. Compare those signals with desired configuration, workload shape and a known healthy baseline before choosing a mitigation. **Trade-off.** The choice normally balances simplicity, isolation, performance, recovery effort, portability and cost; moving a boundary solves one problem while shifting another to the caller or operator. **Production example.** A team operating a service in which Gateway endpoint matters records the workload and configuration baseline, injects one bounded failure in a sandbox, alerts on the first user-impact or saturation signal, and verifies recovery through the original VPC endpoints and AWS PrivateLink journey rather than an administrative green status alone.

### VPC-ENDPOINTS-AND-AWS-PRIVATELINK-JN-04

- [ ] **Question:** What is Endpoint policy, and why does it matter in VPC endpoints and AWS PrivateLink?
> **Covered in:** [VPC endpoints and AWS PrivateLink — Endpoint policy](README.md#endpoint-policy)

**Answer:** **Definition.** Constrains calls through the endpoint but does not grant principal/resource permission. **Mechanism.** At runtime a caller presents or derives an identity, the enforcement point gathers identity, resource and request context, and applicable rules produce allow or deny. The effective decision is the intersection of all guardrails; encryption protects bytes but does not replace authorization, and audit records explain which decision was made. Its guarantee applies only at its own boundary, so upstream identity, downstream dependencies and asynchronous convergence still matter. **Evidence and failure behavior.** Healthy evidence includes a short-lived attributable identity, narrowly scoped access and an audit event for the intended resource. Failures commonly come from expired credentials, mismatched trust, an overriding deny, wrong resource scope or key/certificate lifecycle problems; widening access may hide the cause while creating a breach path. Compare those signals with desired configuration, workload shape and a known healthy baseline before choosing a mitigation. **Trade-off.** The choice normally balances simplicity, isolation, performance, recovery effort, portability and cost; moving a boundary solves one problem while shifting another to the caller or operator. **Production example.** A team operating a service in which Endpoint policy matters records the workload and configuration baseline, injects one bounded failure in a sandbox, alerts on the first user-impact or saturation signal, and verifies recovery through the original VPC endpoints and AWS PrivateLink journey rather than an administrative green status alone.

Correlate identity, desired revision, events, SLI, dependency state and capacity; never make a destructive move from a single green or red field. **Mechanism and risk.** Route-table target for supported services such as S3/DynamoDB without endpoint ENIs.
