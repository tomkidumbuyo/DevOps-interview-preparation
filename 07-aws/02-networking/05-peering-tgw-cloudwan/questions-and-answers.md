### VPC-PEERING-TRANSIT-GATEWAY-AND-CLOUD-WAN-JN-01

- [ ] **Question:** What is VPC peering, and why does it matter in VPC peering, Transit Gateway and Cloud WAN?
> **Covered in:** [VPC peering, Transit Gateway and Cloud WAN — VPC peering](README.md#vpc-peering)

**Answer:** **Definition.** Direct non-transitive connectivity that requires non-overlap and routes on both sides. **Mechanism.** A real request crosses several independent states: name resolution returns an address, the source selects a route and source address, link or overlay forwarding reaches the next hop, stateful or stateless policy evaluates the flow, transport establishes communication, and TLS/application protocols negotiate their own contract. The return path must also work. Its guarantee applies only at its own boundary, so upstream identity, downstream dependencies and asynchronous convergence still matter. **Evidence and failure behavior.** Healthy evidence progresses layer by layer: correct name/address, expected route, permitted flow, listening endpoint, successful handshake and valid application response. Timeouts, refusals, resets and protocol errors mean different layers; packet loss, MTU, asymmetric paths, connection-state exhaustion and proxy timeout mismatch are common production failures. Compare those signals with desired configuration, workload shape and a known healthy baseline before choosing a mitigation. **Trade-off.** The choice normally balances simplicity, isolation, performance, recovery effort, portability and cost; moving a boundary solves one problem while shifting another to the caller or operator. **Production example.** A team operating a service in which VPC peering matters records the workload and configuration baseline, injects one bounded failure in a sandbox, alerts on the first user-impact or saturation signal, and verifies recovery through the original VPC peering, Transit Gateway and Cloud WAN journey rather than an administrative green status alone.

> **Covered in:** [VPC peering, Transit Gateway and Cloud WAN — Transit Gateway attachments](README.md#transit-gateway-attachments)

**Answer:** **Definitions.** VPC peering: Direct non-transitive connectivity that requires non-overlap and routes on both sides. Transit Gateway attachments: Connect VPC, VPN, Direct Connect gateway or peering into a routed hub. **Mechanism.** In VPC peering, Transit Gateway and Cloud WAN, compare where each acts in the request or resource lifecycle, which state it owns, whether its result is synchronous or eventually reconciled, and how callers observe completion.
