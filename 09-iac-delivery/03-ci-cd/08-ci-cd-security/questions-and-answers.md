## Junior — normal conceptual and code questions

### CI-CD-SECURITY-JN-01

- [ ] **Question:** What is OIDC instead of static credentials, and why does it matter in CI/CD security?
> **Covered in:** [CI/CD security — OIDC instead of static credentials](README.md#oidc-instead-of-static-credentials)

**Answer:** **Definition.** The term OIDC instead of static credentials refers to a configuration, state or promotion mechanism that moves a reviewed revision toward effective runtime state within CI/CD security. **Mechanism.** At runtime a caller presents or derives an identity, the enforcement point gathers identity, resource and request context, and applicable rules produce allow or deny. The effective decision is the intersection of all guardrails; encryption protects bytes but does not replace authorization, and audit records explain which decision was made. Its guarantee applies only at its own boundary, so upstream identity, downstream dependencies and asynchronous convergence still matter. **Evidence and failure behavior.** Healthy evidence includes a short-lived attributable identity, narrowly scoped access and an audit event for the intended resource. Failures commonly come from expired credentials, mismatched trust, an overriding deny, wrong resource scope or key/certificate lifecycle problems; widening access may hide the cause while creating a breach path. Compare those signals with desired configuration, workload shape and a known healthy baseline before choosing a mitigation. **Trade-off.** The choice normally balances simplicity, isolation, performance, recovery effort, portability and cost; moving a boundary solves one problem while shifting another to the caller or operator. **Production example.** A team operating a service in which OIDC instead of static credentials matters records the workload and configuration baseline, injects one bounded failure in a sandbox, alerts on the first user-impact or saturation signal, and verifies recovery through the original CI/CD security journey rather than an administrative green status alone.
